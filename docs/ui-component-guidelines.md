# UI 组件与交互规范

本项目使用复制到仓库内的 shadcn/ui Rhea 组件，并以 Base UI 为默认 primitive。规范的目标不是让所有控件完全相同，而是让相同业务层级、相同语义和相同操作风险使用一致的组件、尺寸、状态和反馈。

## 核心原则

1. **优先使用官方 Rhea primitive**，业务代码只组合 `components/ui`；
2. **同一业务维度保持同一种交互**，不要按页面临时变化；
3. **同一视觉层级保持同一表面**，包括高度、圆角、背景、边框、文字对齐和焦点环；
4. **状态必须可见且可访问**，不能只依赖颜色、Tooltip 或隐藏按钮；
5. **桌面与移动端保持相同业务语义**，只改变布局，不创造第二套流程；
6. **后端权限是事实来源**，前端隐藏操作只用于减少干扰。

## Select 与 Popup Combobox

> 固定短枚举使用 Select；实体选择、级联维度和长列表使用 Popup Combobox；同一业务维度必须保持同一种交互组件。

判断优先级：

1. 同一组级联或同一种实体选择必须统一；
2. 实体、动态数据和用户会凭名称查找的内容使用 Combobox；
3. 固定但较长、搜索价值高的列表使用 Combobox；
4. 选项少、含义明确、能一次扫完的枚举使用 Select。

### Popup Combobox

统一复用：

```text
web/src/components/shared/searchable-combobox.tsx
```

要求：

- 触发器使用与 Select 相同的 `bg-input/50` 表面、高度、圆角和焦点环；
- 标签和占位符左对齐，箭头固定在右侧；
- 搜索框位于 Popup 顶部；
- 支持上下键、Enter、Escape 和自动高亮；
- 有明确的无结果状态；
- 搜索只过滤当前有效候选，不得绕过上级条件；
- 上级变化后清理失效的下级值；
- 在 Sheet/Dialog 内，Portal 必须挂载到最近的模态容器；
- 无结果 live region 在有选项时不得占据视觉空间。

当前场景：扫描任务、Agent、大洲、国家/地区、城市、colo、时区。

### Select

Select 用于固定短枚举，例如：

- 最新结果 / 历史记录；
- 可用状态和时间范围；
- 每页数量；
- 采样方式、协议和 Cron 预设；
- 任务、Agent、ASN 和账号状态；
- 黑名单原因和账号角色。

Select 必须通过官方 `items` 映射显示标签，不得把 `true`、`24h`、字段名等内部值暴露给用户。

## 筛选工具栏

标准顺序：

```text
搜索 → 主筛选 → 补充筛选 → 重置 → 显示列
```

规则：

- 搜索、Select、Combobox、重置和显示列统一为 32px；
- Select 与 Combobox 的文字统一左对齐；
- 重置按钮必须常驻，无筛选时可见但禁用；
- 重置必须清除搜索、枚举、级联和分页位置；
- 列管理使用官方 `DropdownMenuCheckboxItem`；
- 列定义必须配置 `meta.label`，不得显示数据库字段名；
- 每个选择控件必须有稳定的可访问名称，不能只依赖当前值；
- 窄屏允许换行，不允许通过缩小文字或横向挤压破坏可用性。

统一重置组件：

```text
web/src/components/shared/data-table/data-table-reset-button.tsx
```

## 按钮与操作层级

- **Primary / default**：页面或表面内唯一主操作，如“创建任务”“保存”；
- **Outline / secondary**：次要但明确的操作，如“取消”“立即运行”“显示列”；
- **Ghost / tertiary**：低强调工具操作，如表格重置、行菜单触发器；
- **Destructive**：删除、停止等高风险操作，只在明确语义和确认流程中使用；
- 同一表面不要出现多个视觉权重相同的主按钮；
- 图标按钮必须提供 `aria-label`；
- 异步提交时禁用重复操作，并显示加载状态；
- 不使用禁用按钮代替权限隐藏：查看者无权执行的服务端写操作应直接不展示。

## 表单

- 所有字段使用可见 Label；
- 错误信息位于字段附近，不只使用 Toast；
- 固定枚举使用 Select，长列表使用 Combobox；
- Switch 用于“立即生效的启停”，不要代替需要保存的普通布尔字段；
- 表单提交按钮放在 Dialog Footer；
- 有未保存修改的 Dialog 在关闭前必须确认；
- 重置自己密码导致当前会话失效时，前端必须返回登录页；
- 不要在 placeholder 中放置唯一的字段说明。

## Dialog、Sheet 与确认

- **Dialog**：创建、编辑、审批和其他需要用户集中完成的业务任务；字段较多时使用更宽的居中 Dialog；
- **Sheet**：只用于补充主界面的非阻断内容，例如移动端 Sidebar 或未来的只读详情检查器；业务 feature 不得直接使用 Sheet 承载表单；
- **AlertDialog**：删除、停止、放弃未保存修改等需要用户明确确认的操作；
- 长 Dialog 使用固定 Header、可滚动 Content、固定且完全不透明的 Footer；
- Dialog 最大高度不得超过视口，移动端保留四周安全间距并让内容区独立滚动；
- Popup 在模态内必须保持正确 Portal、焦点锁和键盘操作；
- 关闭动画期间不得出现可点击但不可见的重复触发器。

## Table 与移动卡片

- 基础 `Table` 保持官方实现；业务分页、排序、空态和边距放在 `DataTable`；
- 大圆角表面首尾列内容使用 16px 边距；
- 排序必须明确是本地还是服务端，不能只对当前页伪排序；
- 总数必须来自真实查询，不能用已加载数组长度代替；
- 空状态 `colSpan` 使用当前可见列数；
- 移动端用卡片表达相同数据语义，保留关键状态、主指标和必要操作；
- 移动卡片不能成为独立业务实现，筛选、权限和操作含义应与桌面一致。

## 页面反馈

- 初次加载使用 Skeleton，不使用空白页面；
- 无数据使用 `EmptyState`，说明当前状态和下一步；
- 可恢复错误使用 `ErrorState` 和重试；
- 成功、失败和后台操作结果使用 Sonner Toast；
- 页面级警告使用 Alert；
- 不可逆操作必须使用 AlertDialog，不只依赖 Toast；
- 运行时错误由全局错误边界接管，并提供重试入口。

## 页面与导航

- 页面标题、描述和主操作统一使用 `PageHeader`；
- 设置等同页分类使用标题下方的单层横向 Tabs；
- 不在全局 Sidebar 外增加占用内容宽度的二级侧栏；
- Sidebar 展开时使用分组，折叠时形成连续图标轨道；
- 折叠 Tooltip 只在主动悬停后显示，不在折叠瞬间批量出现；
- Sidebar Rail 保留宽点击区，但视觉提示只使用居中的短竖线。

## 响应式

- 以业务优先级决定隐藏内容，不按桌面布局机械缩放；
- 移动端 Header 保持 48px；
- Tabs 和筛选可横向滚动或换行，但页面本身不得横向溢出；
- Dialog、移动 Sidebar Sheet 和 Popup 不得超出视口；
- 触控目标不能因紧凑视觉小于可操作区域；
- 桌面浅色与移动深色是永久浏览器回归的最低覆盖。

## 图标

项目统一使用 Lucide：

- 默认描边 `1.75`；
- 文字控件图标 `16px`；
- 紧凑辅助信息 `14px`；
- 卡片标题和空状态 `20px`；
- 品牌或独立加载状态最多 `24px`，必须有明确语义；
- 图文间距通常为 `gap-1.5` 或 `gap-2`；
- 不依赖 Lucide 默认 `24px`；
- 不只通过放大或加粗图标表达选中状态。

## 可访问性

- 所有交互必须支持键盘；
- 图标按钮和无可见 Label 的触发器必须有稳定名称；
- 焦点状态不能被自定义样式移除；
- 颜色不是唯一状态信号；
- Tooltip 只提供补充信息，核心内容必须直接可见；
- 表单错误、空状态和异步状态应能被辅助技术识别；
- 模态打开后焦点进入内容，关闭后返回合理触发器。

## 禁止事项

- 不把所有 Select 一刀切成 Combobox；
- 不在 Select 中自行塞搜索框；
- 不使用浏览器原生交互替代项目 UI primitive；
- 不在业务页面临时创造按钮高度、圆角、阴影或图标规格；
- 不把内部变量、枚举值和数据库字段名展示给用户；
- 不通过条件挂载导致工具栏布局跳动；
- 不为了“看起来禁用”省略后端权限校验；
- 不直接覆盖官方组件而丢失本地适配。

## 新场景示例

| 场景 | 组件 | 原因 |
|---|---|---|
| 两种账号角色 | Select | 固定短枚举 |
| 数百个用户中选负责人 | Combobox | 实体选择，需要搜索 |
| 时区 | Combobox | 固定但长，搜索价值高 |
| 五个时间范围 | Select | 固定短枚举 |
| 四级地理筛选 | Combobox | 同一套级联维度 |
| 删除账号 | AlertDialog | 不可逆操作 |
| 编辑扫描计划 | Dialog | 集中完成的长表单，使用固定标题、滚动内容和固定操作区 |
| 重置密码 | Dialog | 短表单 |

新增场景无法明确判断时，优先保持同类业务组件一致，并在本文件记录最终决定。
