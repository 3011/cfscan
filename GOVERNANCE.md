# Governance

CF Scanner currently uses a maintainer-led governance model.

The repository owner and listed code owners make final decisions on roadmap, architecture, releases, security response, and contributor permissions. Decisions should be explained in issues or pull requests when they affect public behavior.

Long-running or high-impact changes should begin with a public issue. Maintainers may request an RFC-style proposal covering motivation, alternatives, compatibility, migration, security, and testing.

As the contributor base grows, maintainership can be expanded based on sustained high-quality contributions, review participation, security judgment, and respectful community conduct.
